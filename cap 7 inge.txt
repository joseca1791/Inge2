Capitulo 7

Arquitectura de diseño:
la o las estructuras del sistemas que
abarcan elementos de software, propiedades
externas visibles para esos elementos y las 
relaciones entre ellos.

diseño detallado: los componentes son descompuestos
a un nivel mas detallado.
se necesitan la arquitectura de diseño junto con los
requerimientos bien refinados para producir el diseño detallado

vista: una representacion de la estructura de un sistema

vista logica: representa la orientacion de objetos
descompuestos de un sistema, clases y relacion entre
ellos. (UML)

vista de proceso: representa los componentes en tiempo
de ejecucion y como interactuan entre ellos.

Tambien
-vista de subsistema descompuesto
-vista de la arquitectura fisica
-vistas de modulo
-vista de tiempo de ejecucion
-vistas de asignacion

conocimiento meta arquitectura: diferentes maneras para comparar
o escoger architecturas para un sistema, para la hora
de tener que escoger una para empezar

tres tipos: 
-stilos o patrones [cliente-servidor, MVC]
-tactitas [resuelve problemas menores, mas epecificos]
-arquitecturas referenciadas [conjunto de las principales
desiciones que son aplicadas simultaneamente a varios sistemas]

sirve para: punto de partida para ña arquitectura de un sistema
y es un mecanismo efectivo de comunicacion para proveer una idea
rapida de una estructura de alto nivel de un sistema

descomposicion funcional: separar la funcion en pequeños
modulos, que unidos forman un unico modulo
